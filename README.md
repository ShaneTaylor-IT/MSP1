### The Why
------
The primary goal of this website is to give viewers some insight into Fullys Fresh Food, showcase the different offerings available, to advertise new sunday roasts for delivery, to be able to inquire for a quote for cetering events. The aim is to attract new customers into the local shop by listing the offerings available and showcasing a menu page, also to advertise new Sunday roast dinners available for delivery, and to request quotes for catering events. The website is designed to provide information and build an online prefessional presence for the business.

### Business Goal
------
* Increase customer awareness of the offering available.
* Provide a way for customers to request a quote for catering events.
* Showcase new Sunday roasts with emphasis on new *delivery* option.
* Provide a professional online presence for the business.

### (UX) User Experience
------
Users will be able to navigate the website easily to clearly view the offerings, new advertised Sunday roasts, request quotes and, view the menu. The quotaion form will be simple but also allow for all relevent information to be collected to make requests easier.

### Design
------
The design for this website is a mobile first approach, modern and easily accessable. The website uses colours that complement the feel for a friendly, welcoming food establishment.

#### Design brief
------

* Colours
   * Green and Cream to mimick the feel of some of the fresh foods available for sale.
   * Blue and Orange to show important navigation and highlights.

* Font
   * Paprika - To give headers a hand written feel
   * Roboto - To give text a clear easy to read style

* Images
   * Hero image to match the colours of the website
   * A variety of images showcasing owner, offerings and products

### Features
------
#### Existing Features
* Navigation Bar: Links to Home, Contact, Offerings, About, Event Hire and Join the Team
* Contact Form: Allows customres to easily request quotes
* Social Media Links: Connect with the company on Facebook, Instagram and more
* Menu Page: Showcasing products available in store and for delivery
* Testimonial Section: Ability for customers to see reviews left by other customers
* Contact information: Address, phone number, email address and opening time are all clearly displayed in the footer of each page.

#### Future Features
* Online Booking: A way for customers to place orders online and direct payment on the website.
* Service cards to lead not just to menu page but to each of the nav pills. So when clicking on Deli service card menu button it will display the Deli menu within menu.html.
* Add a gallery page after collecting professional photos of the establishment, facilities, products and staff.
* Add full price range on all products within the menu.

### Issues and bugs
------
* **Testimonies** - One challenge I came across was how to create the testimonies, after some searching on google I found it was very simple to use bootstrap card layout and mix it with the carousel.
   * Also to add more detail to the testimonies was to add a star rating which was using the star icon from Font Awesome. ```HTML - <i class="fa-solid fa-star"></i>```
* **Menu Nav Pills** - A challenge when creating the menu was styling the buttons on the pill menu layout, took quite some time to research and figure out which bootstrap classes where styling the color for the active pill. With the help of dev tools I was able to eventually identify the classes involed and then correctly style them.
   * Correctly identified with the use of dev tools and [bootstrap documentation](https://getbootstrap.com/docs/5.3/components/navs-tabs/#working-with-flex-utilities)
* **Testimonails carousel sizing** - A issue I came acroos was with the testimonials as the carousel would cycle through the testimonies there would be a page layout issue on smaller screens where some testimonials where larger in text size cause the rest of the page to move down to accomidate the size and then revert back with smaller testimonials. To get around this I added a Media queries in the CSS style to give all testimonials a fixed height.
